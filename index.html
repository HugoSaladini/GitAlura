    • -------------- Comandos do Git ----------------
    README.md
    git init (vai inicializar um repositorio git dentro da pasta)
    git status (verifica o status dos arquivos)
    git add . (adiciona todos os aquivos pendentes)
    git commit -m "Mensagem de commit" (realiza o commit dos arquivos add)
    git remote add origin ..git@hub..
    git push -u origin master

    ls \\ Mostra os arquivos dentro da pasta
    git log \\ Mostra historico de commits em detalhes
    git log --oneline \\ Mostra históricos de commits de forma resumida
    git log -p \\ Mostra as alterações dos commits apertandoa a letra "q" você ira sair do log -p

    -------------- CONTROLES DE BRANCH ----------------
    Andes de começar a atividade, verificar se a main/master esta atualizada.
    git pull (local do servidor/origin normalmente) master/main
    git branch \\ Checa as branch existentes
    git branch (nome da branch) \\ cria a branch com o nome escolhido
    git checkout (nome da branch) \\ Entra dentro da branch escolhida
    git checkout -b (nome da Branch)\\ Cria uma nova Branch com o nome escolhifo, e já entra dentro da Branch
    git merge \\ Unifica o trabalho de duas Branchs separadas em um unico local
    git rebase (nome da branch) \\ Atualiza a branck onde você esta trabalhando, com o branch do nome escolhigo

    git checkout -- (nome de arquivo que deseja desfazer)

    normalmente a opção neste caso aparece no console:
    git reset HEAD ( nome de arquivo )
    Feito isso as informações ainda estaram no arquivo, com isso voce pode voltar a editar ou desfazer o arquivo

    deve saber o rash do commit gerralmente os 5 primeiros carctrs ja bastm se necessario utilizar git log para ver as informações

    :x para sair do vint

    ------GUARDANDO PARA FAZER DEPOIS
    git stash
    vai salvar minhas modificações sem gerar um commit, gerando rash da aqual posso voltar se necessário
    git stash list
    mostra a lista de stash que estao disposniveis
    git stash apply {0} passar apenas o número sem os conchets
    e é necessário realizar o
    git stash drop para retirar o stash que foi modificado
    git stash pop ele faz o merge com as modificações e ja exlui o aquivo stash
